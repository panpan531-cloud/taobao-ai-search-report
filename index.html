01 / SOCIAL SIGNALS

用户不是不想搜，而是不再确定搜索结果是否服务于自己的任务

小红书站内深抓在本次运行中触发验证码和浏览器扩展限制，因此这里采用“方向性社媒信号 + 公开可索引讨论 + 行业资料”的交叉归纳。结论不声称代表全量评论，但足以形成产品判断：搜索体验的问题是信任、控制感和决策效率。

要点整理
相关性被稀释

用户输入明确商品词，却感到结果被近期浏览、推荐流、广告位或泛类目热品影响；搜索页被感知为“另一个推荐页”。

同质化让选择成本上升

服饰、美妆、家居等类目中，大量相似标题和图片堆叠，差异点不透明，用户需要自己判断材质、版型、店铺可信度。

精确找同款仍有断点

用户从截图、博主内容、旧订单、线下场景回到淘宝时，往往缺少完整商品名，传统搜索难以恢复上下文。

价格和优惠难以一眼判断

券、满减、88VIP、跨店活动、运费和退换货规则叠加后，用户更想知道“现在买是不是划算”。

图片找同款心智稳定

当用户有截图、商品图或穿搭参考时，淘宝仍是高频回流入口，说明多模态搜索基础可继续放大。

长尾供给仍是优势

淘宝的 SKU 覆盖广、替代品多、非标品丰富。AI 的价值是把供给广度变成可理解的选择结构。

交易链路天然可闭环

搜索、领券、下单、物流、售后都在同一体系内，AI 导购可以从推荐延伸到价格计算和履约风险提示。

从评论到产品问题
低决策成本 高决策成本
相关性不是单纯排序问题

当搜索结果混入推荐和商业化信号，用户会怀疑入口中立性。AI 搜需要先识别“任务搜索”，再把推荐信号降权或解释化。

02 / QUERY STRUCTURE

淘宝 query 可以分成七类，AI 应优先处理“约束、场景、决策”三类复杂意图

分布是工作假设：参考电商搜索 query 类型研究、传统搜索意图框架，以及淘宝商品搜索的交易属性推导。后续可以用真实 query log 校准份额，但不影响先判断 AI 搜的首批落地点。

判断原则

越是“用户说不清商品名、需要综合约束、需要比较和解释”的 query，越适合由 AI 导购接管。

03 / AI SHOPPING GUIDE SCENARIOS

AI 导购要落在“商品理解 + 个人上下文 + 交易计算”交汇处

仅把搜索框换成问答框，容易变成低频工具。更值得优先做的是嵌入搜索结果、商品详情、活动页和订单链路的 AI 能力，让用户少筛选、少比价、少踩坑。

04 / PRODUCT RECOMMENDATION

建议用“AI 搜三层架构”推进：先修复搜索信任，再扩大导购场景

L1

意图识别层

识别精确、类目、约束、场景、对比、售后、内容种草七类 query；对任务搜索降低推荐污染。

L2

商品理解层

抽取材质、版型、尺寸、功效、适用人群、价格、履约和评价风险，形成可比较的商品卡。

L3

导购决策层

用解释式排序、对比表、到手价计算、清单生成，把“搜到商品”推进到“做出购买判断”。

05 / EXECUTION ROADMAP

从低风险入口切入，三阶段验证 AI 搜的增量价值

首批不宜追求全站万能助手，而应选择“传统搜索高摩擦、AI 解释价值高、交易指标可闭环”的场景做灰度。

Phase 1

搜索页 AI 摘要与重复合并

服饰/家居/数码等高比较类目先上线：相似款聚类、差异点摘要、到手价提示。

验证：筛选减少、加购提升、负反馈下降
Phase 2

约束式导购与对比表

支持自然语言约束，生成可编辑对比表，并保留可解释证据和商详跳转。

验证：对比使用率、购买转化、退货率
Phase 3

任务型清单与个人上下文

引入订单、收藏、尺码偏好、地址和会员权益，形成搬家、旅行、开学等清单式购物。

验证：多件购买、复访、用户信任评分
Search Quality 相关点击率

搜索后首屏有效点击、重复改 query 次数、筛选使用深度。

Decision Quality 导购采纳率

对比表打开、AI 摘要展开、推荐理由点击、收藏/加购。

Transaction Quality 有效成交

支付转化、售后申请、退货率、评价风险命中。

Trust 可控感

不喜欢/屏蔽/举报下降，用户对“推荐理由”的正反馈。
